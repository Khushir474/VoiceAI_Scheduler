# DOPS-18: Max handles off-calendar event updates with follow-up questions and contextual suggestions

When user tells Max about an event not on their calendar during the call (e.g. 'I have a dinner tonight in Midtown' or 'I'm meeting a friend at 5'), Max should: (1) ask 2-3 targeted follow-up questions to gather missing details (location if missing, duration, indoor/outdoor, dress code if relevant); (2) fetch real-time weather for that location/time; (3) calculate commute from user's current/last known location; (4) generate contextual suggestions: when to leave, what to wear/bring (umbrella, jacket, etc.), buffer time. Max optionally offers to add the event to calendar (DOPS-8). Implementation: conversation_agent detects 'unscheduled event' intent from user utterance; triggers follow-up question flow (2-3 questions max, non-repetitive); after gathering details, calls weather tool (DOPS-11) and maps adapter for commute; synthesizes leave time + carry items recommendation; optionally calls calendar write (DOPS-8) after user confirms. Touches: agents/conversation_agent.py, agents/prompts.py, adapters/weather.py, adapters/maps.py. Acceptance: Correctly identifies off-calendar event mentions; asks ≤3 follow-up questions; fetches weather for event location and time; calculates commute; gives leave time and carry items suggestions; offers calendar add; all steps logged to debug_logs.
