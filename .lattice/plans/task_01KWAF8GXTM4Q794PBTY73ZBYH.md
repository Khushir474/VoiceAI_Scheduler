# DOPS-23: Bidirectional text conversation — user can text Max and Max replies in context

Text-channel equivalent of DOPS-17 (voice bidirectional conversation). User can text Max via iMessage or SMS after the call ends (or outside of a call) and Max replies within his daily planning role. Max maintains context of the day's plan and call transcript when responding. Examples: 'Hey Max, what time should I leave for my 3pm?' → Max calculates commute and replies; 'Add yoga at 7am tomorrow' → Max confirms and creates calendar event (DOPS-8); 'What's the weather like tonight?' → Max fetches and replies (DOPS-11). Implementation: inbound SMS/iMessage webhook receives user message; routes to conversation_agent with current AgentState + call transcript as context; LLM generates reply within Max's persona and planning scope; reply sent back via same channel (iMessage bridge or Twilio). Touches: api/messages.py (or new inbound handler), agents/conversation_agent.py, adapters/messaging/imessage_bridge.py, adapters/messaging/twilio_sms.py, agents/state.py. Acceptance: User can text Max post-call and receive a reply; Max references day's plan in responses; multi-turn text thread works; Max stays in planning scope; all exchanges logged to debug_logs; works on both iMessage and Twilio channels.
