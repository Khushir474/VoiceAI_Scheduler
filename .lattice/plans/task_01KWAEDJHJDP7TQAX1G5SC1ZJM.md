# DOPS-21: Clean up summary message formatting and content

The end-of-call iMessage/SMS summary needs a content and formatting cleanup. Current _format_plan_for_sms() output is functional but likely cluttered, inconsistent, or hard to scan on a phone screen. Audit and fix: (1) remove redundant or verbose sections; (2) ensure consistent emoji usage and section headers; (3) tighten copy — each line should be scannable in 2 seconds; (4) logical ordering (time-ordered events, then leave time, then carry items, then workout); (5) handle edge cases cleanly (no events, no commute, virtual-only day); (6) test on real phone screen for readability. Touches: agents/conversation_agent.py (_format_plan_for_sms), agents/prompts.py. Acceptance: Summary fits comfortably on a phone screen without scrolling for a typical day; sections are clearly separated; no duplicate info; edge cases render cleanly; reviewed against a real SMS output.
