# DOPS-19: Bug: Max says morning greeting/scheduler intro twice on call pickup

Max is greeting the user and saying the morning scheduler intro twice — once on call reception (Vapi assistant-speaks-first trigger) and once again after the user responds. Root cause likely: (1) firstMessageMode is set to 'assistant-speaks-first' in vapi.py initiate_call() which triggers an auto-greeting, AND (2) the system prompt or conversation_agent also sends a greeting message as the first LLM turn, causing a duplicate. Investigation: check vapi.py assistantOverrides.firstMessageMode, check dailyops_prompt.py for hardcoded greeting, check conversation_agent.run() for any explicit greeting send before the plan delivery, check Vapi assistant config on dashboard for a firstMessage field. Fix: ensure greeting happens exactly once — either via Vapi firstMessage (no LLM call needed) OR via conversation_agent first turn (disable Vapi auto-greeting), not both. Touches: adapters/voice/vapi.py, adapters/voice/dailyops_prompt.py, agents/conversation_agent.py. Acceptance: User hears Max greet exactly once on pickup; no duplicate intro; verified via Vapi call log and debug_logs transcript.
