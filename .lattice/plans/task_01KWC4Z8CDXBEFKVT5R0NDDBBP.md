# DOPS-24: Pre-call location resolution: current location, work default null, event locations indexed

Before initiating the morning call, resolve all location data needed for commute and traffic calculations. Three location sources: (1) Current location — fetch user's current location before call (device GPS via a lightweight endpoint, or fallback to home address from user_preferences); (2) Work location — default null unless set in user_preferences; agent should not assume user is going to work unless explicitly configured or a work calendar event exists; (3) Event locations — extract and index the location field from all upcoming calendar events for the day so the planning agent can look up any event's location without re-parsing. Pre-call resolution means all locations are available in AgentState before the call starts, so commute calculations (DOPS-9) and traffic fetches (maps adapter) happen without mid-call delays. Implementation: add location_context to AgentState (current_location, work_location, event_locations dict keyed by event_id); planning_agent resolves all three before call; work_location defaults to None if not set; maps adapter uses these for commute calculation. Add user_preferences fields: home_address, work_address (nullable). Touches: agents/planning_agent.py, agents/state.py, migrations/ (new preference fields), adapters/maps.py. Acceptance: AgentState.location_context populated before call starts; work_location is None by default; event_locations has location string for each event that has one; commute calculations reference this index; all location resolution logged to debug_logs.
