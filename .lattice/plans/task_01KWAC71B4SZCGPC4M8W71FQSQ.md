# DOPS-15: Max full-day calendar context with 1-day awareness window

Max should have full context of all calendar events for the present day (00:00–23:59 ET), including events that have already finished. This gives Max awareness of what the user did earlier in the day, not just what's upcoming. Awareness window is strictly 1 day — no events from yesterday or tomorrow bleed in. Implementation: planning agent fetches all events for current calendar day in ET timezone (midnight to midnight); includes past events in AgentState with a 'completed' flag or similar marker; conversation agent receives full day context; Max can reference earlier events naturally ('You had a standup this morning — how did that go?'). Touches: agents/planning_agent.py, agents/state.py, adapters/calendar/google_calendar.py, adapters/calendar/apple_ical.py. Acceptance: All same-day events (past + upcoming) included in AgentState; timezone anchor is ET (America/New_York); no cross-day bleed; past events distinguishable from upcoming ones; Max can reference earlier events in conversation.
