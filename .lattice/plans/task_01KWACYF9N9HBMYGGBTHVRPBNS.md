# DOPS-17: Bidirectional conversation — user can respond to Max and Max replies in context

Currently Max delivers a briefing monologue. This ticket enables true back-and-forth: user can respond to anything Max says and Max will reply naturally within his role as a daily planning assistant. Max maintains conversation context across turns, references prior exchanges, and stays in character. Examples: user says 'actually I moved that meeting to 3pm' → Max acknowledges and updates the plan; user asks 'should I bring an umbrella?' → Max checks weather context and answers; user says 'can you add a gym session at 6pm?' → Max confirms and triggers calendar write (DOPS-8). Implementation: wire Vapi transcript streaming (VapiWebSocketClient already implemented) into conversation_agent turn loop; conversation_agent processes each user utterance as a new turn with full prior context; LLM receives full conversation history + AgentState; Max responds appropriately per role (planner, not general assistant). Touches: agents/conversation_agent.py, adapters/voice/vapi.py, adapters/voice/vapi_websocket.py, agents/state.py, agents/prompts.py. Acceptance: User can speak during Max's briefing and Max responds; multi-turn conversation works end-to-end; Max stays on-topic (daily planning scope); conversation history persists across turns within call; all turns logged to debug_logs with speaker, utterance, and response.
